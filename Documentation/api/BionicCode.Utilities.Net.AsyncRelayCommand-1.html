    Class AsyncRelayCommand<TParam>

    A reusable command that encapsulates the implementation of System.Windows.Input.ICommand with support for async/await command delegates.
    Enables instant creation of an ICommand without implementing the ICommand interface for each command. The AsyncRelayCommand<TParam> accepts asynchronous command handlers and supports data bindng to properties like by implementing System.ComponentModel.INotifyPropertyChanged.
    Call and await the BionicCode.Utilities.Net.Common.IAsyncRelayCommandCommon.ExecuteAsync() method or one of its overloads to execute the command explicitly asynchronously. System.Windows.Input.ICommand

    IsCommandManagerRequerySuggestedEnabled

    Controls whether the command's System.Windows.Input.ICommand.CanExecuteChanged is attached to the System.Windows.Input.CommandManager.RequerySuggested event.

    Declaration
    public bool IsCommandManagerRequerySuggestedEnabled { get; set; }
    Property Value
    Type Description
    System.Boolean

    true to attach the System.Windows.Input.ICommand.CanExecuteChanged is attached to the System.Windows.Input.CommandManager.RequerySuggested event.
    The default is true.
    Remarks

    If the IsCommandManagerRequerySuggestedEnabled property is set to true, calling the System.Windows.Input.CommandManager.InvalidateRequerySuggested method will also raise the System.Windows.Input.ICommand.CanExecuteChanged event.
    This is the default behavior. System.Windows.Input.CommandManager.InvalidateRequerySuggested is regularly invoked by the framework to keep UI elemnts, that implement System.Windows.Input.ICommandSource (like the Button control) and listen to the System.Windows.Input.ICommand.CanExecuteChanged event, updated.
    When the IsCommandManagerRequerySuggestedEnabled property ios set to false, the System.Windows.Input.ICommand.CanExecuteChanged must be raised explicitly by calling the method.
