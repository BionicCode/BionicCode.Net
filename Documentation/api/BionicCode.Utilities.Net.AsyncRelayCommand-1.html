    Class AsyncRelayCommand<TParam>

    A reusable command that encapsulates the implementation of ICommand with support for async/await command delegates.
    Enables instant creation of an ICommand without implementing the ICommand interface for each command. The AsyncRelayCommand<TParam> accepts asynchronous command handlers and supports data bindng to properties like IsExecuting by implementing INotifyPropertyChanged.
    Call and await the ExecuteAsync() method or one of its overloads to execute the command explicitly asynchronously. ICommand

    IsCommandManagerRequerySuggestedEnabled

    Controls whether the command's CanExecuteChanged is attached to the RequerySuggested event.

    Declaration
    public bool IsCommandManagerRequerySuggestedEnabled { get; set; }
    Property Value
    Type Description
    Boolean

    true to attach the CanExecuteChanged is attached to the RequerySuggested event.
    The default is true.
    Remarks

    If the IsCommandManagerRequerySuggestedEnabled property is set to true, calling the InvalidateRequerySuggested() method will also raise the CanExecuteChanged event.
    This is the default behavior. InvalidateRequerySuggested() is regularly invoked by the framework to keep UI elemnts, that implement ICommandSource (like the Button control) and listen to the CanExecuteChanged event, updated.
    When the IsCommandManagerRequerySuggestedEnabled property ios set to false, the CanExecuteChanged must be raised explicitly by calling the InvalidateCommand() method.
