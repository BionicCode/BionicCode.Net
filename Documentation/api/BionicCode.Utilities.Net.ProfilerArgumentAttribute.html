    Remarks

    To start profiling create a configurator by calling CreateProfilerBuilder<TTarget>(). This returns a ProfilerBuilder for a particular Type that allows the configuration of the profiling session.
    Finalize the step by calling RunAsync() to start the profiling for the current type.

    A member can be decorated with multiple ProfilerArgumentAttribute attributes. This is useful to emulate real-world usage where the input is dynamic and may impact the perfomance of the profiled member.
    Each ProfilerArgumentAttribute results in an profiling iteration. If the configured iterations of the ProfilerBuilder exceed the number of ProfilerArgumentAttribute attributes, then the profiler will alterante between the argument lists.
    The default number of iterations is 1 and can be customized by calling SetIterations(Int32).
